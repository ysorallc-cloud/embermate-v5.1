2. **Appointment Information**
   - Healthcare provider names and specialties

3. **Photos & Documentation**

4. **Care Coordination Data**
   - Patient name (for care briefs and reports)
   - Caregiver names and contact information
   - Emergency contact details
   - Family sharing invite codes
   - Care activity logs (who performed what actions)

5. **App Settings & Preferences**
   - Notification settings
   - Security preferences (biometric auth, PIN)
   - App configuration

## Data Storage & Security

### Encryption

All sensitive health data is encrypted using **AES-256 encryption** with authenticated encryption (HMAC-SHA256):

- **At Rest**: All data stored on your device is encrypted
- **In Keychain**: Authentication credentials stored in secure device keychain
- **In Transit**: HTTPS encryption for any future cloud sync (optional feature)

### Biometric Security

- Face ID / Touch ID / Fingerprint authentication
- 4-6 digit PIN fallback
- Auto-lock after 5 minutes of inactivity
- Session management with secure tokens

### Access Control

- Only you can access your data (with biometric auth or PIN)
- Family members you invite have granular permissions (view/edit/export)
- All access is logged in activity feed

## Data Sharing (User-Controlled Only)

You control all data sharing. EmberMate allows sharing **only when you explicitly choose to**:

### Family Sharing
- Generate invite codes to add family members/caregivers
- Set permissions per caregiver (view, edit, mark medications, etc.)
- Revoke access anytime
- View activity log of all caregiver actions

### Care Report Exports
- Generate PDF reports for healthcare providers
- Share via email, messaging, or print
- Reports include: medications, vitals, symptoms, appointments
- **Recommendation**: Password-protect exported PDFs

### Photo Sharing
- Share individual medication or wound photos
- No automatic photo syncing to cloud services
